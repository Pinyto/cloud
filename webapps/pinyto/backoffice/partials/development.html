Pinyto entwickeln

Pinyto ist freie Software, lizensiert unter der GPL v.3. Du kannst Pinyto beliebig kopieren, weiter geben und modifizieren, solange deine geänderte Version auch unter der GPL steht.

Pinyto ist in Python geschrieben und nutzt das Django Framework. Für die Speicherung der in Pinyto gespeicherten Daten wird Mongodb verwendet. Du brauchst zusätzlich eine SQL-Datenbank, in der Benutzerdaten und Quelltexte gespeichert werden. Diese Webseite ist mit Angular geschrieben, was auch für Webapps, die Pinyto benutzen ein empfehlenswertes Framework ist.

Installation

Um Pinyto zu installieren brauchst du einen Server mit Python- und Django-Unterstützung. Auch wenn es gelegentlich Anbieter gibt, die Webspace anbieten, der Django Support bietet, wirst du in den meisten Fällen einen eigenen virtuellen oder Root-Server brauchen auf dem Django installiert werden kann.

Wir empfehlen ein Debian-basiertes System wie Ubuntu, da die Installation und Wartung auf diesen Systemen getestet wurde und gut läuft. Natürlich kannst du Pinyto aber auf jedem System installieren, das die Anforderungen erfüllt:

  • Du brauchst ein Betriebssystem mit seccomp-support im Kernel. Linux bietet diesen Support mindestens seit 2009, bei BSD oder Windows-Kerneln kann es aber gut sein, dass seccomp nicht verfügbar ist. Pinyto wird auf diesen Systemen nicht laufen, da seccomp genutzt wird um die Pinyto-Sandbox auszuführen.
  • Nginx oder Apache wird als Webserver gebraucht um eingehende Requests zu handeln und statische Dateien auszuliefern. Die Voraussetzungen dafür erfüllen die meisten Webserver seit langer Zeit, was bedeutet, dass die stabile Version deines Betriebssystems ausrechen wird. Wenn du jedoch vor hast SPDY oder gar HTTP 2.0 einzusetzen wirst du vermutlich neuere Versionen brauchen.
  • Du brauchst eine SQL-Datenbank für Pinyto. Für eine einfache Installation kannst du SQLite verwenden. Für Installationen mit vielen Usern empfehlen wir PostgreSQL. Prinzipiell arbeitet Pinyto mit allen SQL-Datenbanken, die von Django unterstützt werden. Eine genauere Beschreibung findest du in der Django Dokumentation.
  • Django 1.7 oder neuer. Du brauchst einen wsgi-wrapper, um Requests vom Webserver an Django weiter zu reichen. Wir empfehlen uWSGI für diesen Zweck. Eine Installationsanleitung dafür findest du hier.
  • MongoDB 2.4.9 oder neuer
  • Python 3.4 oder neuer
  • PyMongo

Aufsetzen eines virtualenv

Wir empfehlen ein virtualenv zu benutzen, um mit der Installation keine Systembibliotheken zu stören. Das Virtualenv wird erstellt mit virtualenv venv -p /usr/bin/python3. Danach kann es mit source venv/bin/activate aktiviert werden. Die folgenden Dependencies installiert man anschließend am leichtesten mit pip3 install ...:
  • django
  • cryptography
  • django-cors-headers
  • cffi
  • pymongo
  • beautifulsoup4
  • pytz
  • python-prctl
  • requests
Diese letzte Dependency ist optional, da sie nur gebraucht wird, wenn man die Dokumentation selbst erstellen will:
  • sphinx_rtd_theme

Bei Pinyto mithelfen

Die Grundstruktur von Pinyto steht und es gibt auch einige automatische Tests, die immerhin über 80% des Codes abdecken. Da Pinyto aber als sichere Umgebung für persönliche und wertvolle Daten gedacht ist, muss sowohl die Testabdeckung als auch die sonstige Qualität des Codes noch weiter verbessert werden. Qualität und Wartbarkeit des Codes nehmen wir ernster als möglichst schnell neue Features zu entwickeln.

Pinyto wird version 1.0 erreichen, wenn folgende Features funktionieren: (diese Features sind fertig)

  • Abspeichern beliebiger Daten der Nutzer in schemafreier, durchsuchbarer und verarbeitbarer Form
  • Nutzer können eigenen Code für die Verarbeitung ihrer Daten in einer abgesicherten Sandbox schreiben und auf dem server ausführen
  • Assemblies mit fertigem Code für bestimmte Anwendungsfälle können einfach installiert und auch einfach entfernt werden
  • Einfache übersicht über den verwendeten Speicherplatz und die verbrauchte Rechenzeit
  • Sichere authentifizierung für Webapps und Programme (hier ohne Passwort)
  • Rechtekontrolle, die es Nutzern ermöglicht bestimmte Daten für andere Nutzer freizugeben und bearbeitbar zu machen
  • Eine synchronisierte ToDo-Liste für mehrer Geräte (Desktop, Smartphone, Web)
  • Kontakte (besonders die auf Android Smartphones) können mit Pinyto verwaltet werden
  • Ein synchronisierter Kalender mit rechtzeitigen Benachrichtigungen für Termine
  • Ein synchronisierter Messenger, der Kurznachrichten mit E-Mail in einer Oberfläche verbindet

Jedes dieser Projekte kann deine Unterstützung gebrauchen. Wir planen Linux desktops und Android Smartphones bei den oben erwähnten Apps zu unterstützen aber es wäre wunderbar, wenn jemand Pinyto auch auf Windows, Mac or iOS bringen würde. Auch wenn das Backend open-source ist, müssen die Apps nicht zwingend freie Software sein. Der Dämon, der die Verbindung zur Cloud unterhält, sollte allerdings open-source sein, um das Vertrauen in das Gesamtsystem nicht zu belasten.

Bitte kontaktiere uns, wenn du eine App für Pinyto entwickelst. Wir setzen gerne einen Link auf die Projektseite.

Wenn du bei der Entwicklung des Backends oder einer existierenden App helfen willst, solltest du dich am Besten auf der Mailingliste anmelden.

Developing Pinyto

Pinyto is free Software licensed under the GPL v.3. You are free to distribute and modify Pinyto to match your needs. Pinyto is written in Python using the famous Django framework. Mongodb is used to store the user's data but for administrative purposes there is also a SQL Database needed. This Interface is created with Angular which proved itself very handy for developing webapps using Pinyto.

Installation

For installing Pinyto you need a server which runs Python and Django. There are some hosters which offer webspace where this is possible but mostly you will need your own virtual- or root-server. We recommend using a debian-based system for this server like Ubuntu but it can run on any systems which meets the requirements:

  • You will need seccomp-support in your kernel. Linux has that since 2009 but BSD and Windows-Kernel probably do not have this feature and will not be able to run the Pinyto sandbox.
  • Nginx or Apache for serving static files. There are no complicated requirements so the stable version of your system is fine. However if you plan to use SPDY or HTTP 2.0 you may need a more recent version.
  • You need a SQL database for Pinyto. You may use SQLite which is file based and very easy to install. For installations with a lot of users PostgreSQL is recommended. For a full description of which database can be used see the Django documentation.
  • Django 1.7 or newer. You need a wsgi-wrapper to pass requests to Django. We recommend using uWSGI. Installation instructions for this setup are here.
  • MongoDB 2.4.9 or newer
  • Python 3.4 or newer
  • PyMongo

Setting up a virtualenv

We recommend using a virtualenv which can be created with virtualenv venv -p /usr/bin/python3. After activating the virtual environment with source venv/bin/activate you can install the following dependencies with pip3 install ...:
  • django
  • cryptography
  • django-cors-headers
  • cffi
  • pymongo
  • beautifulsoup4
  • pytz
  • python-prctl
  • requests
This is optional. You only need this theme if you want to build the documentation yourself:
  • sphinx_rtd_theme

Contributing

The core functionality is working and there are some automatic tests which cover more than 80% of the code. Since Pinyto is meant to be a safe place to store precious personal data additional efforts to improve test coverage and overall code quality should be made.

Pinyto will reach version 2.0 if the following features are all working: (those are done)

  • Saving of user data in a schema-free, easily queryable form
  • Users can write own code in a secure sandbox which is executed at the server
  • Assemblies with pre-packaged code for certain use cases can easily be installed and removed
  • Monitoring of used disk space and computation time
  • Secure authentication for webapps and apps for devices (on the device without password)
  • Permission management enabling users to make data accessible and editable for others
  • A cloud synced ToDo-list for different devices (desktop, phone, web)
  • Contacts (especially the ones on Android phones) can be managed through Pinyto
  • A cloud synced calendar which notifies the user on time for dates
  • A synced messenger which combines instant messages with mail in one place

Feel free to contribute to any of these projects. We are planning to support Linux desktops and Android smartphones but it would be great if somebody brought Pinyto to Windows, Mac or iOS. Even if the backend is open-source the apps for the devices do not need to be free. However the daemon handling the connections to the cloud should probably be open-source so users can trust that they do not do something funny.

Please contact us if you are developing an app using Pinyto. We would like to link to your project page.

If you want to help with the backend or any of the existing apps please subscribe to the mailing-list.
